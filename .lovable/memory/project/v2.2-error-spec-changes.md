## What Was Done (v2.2.0)

### Convenience Constructors Added
- `UrlError` / `WrapUrlError` — URL-related errors with automatic context
- `SlugError` / `WrapSlugError` — Slug validation errors
- `SiteError` / `WrapSiteError` — Site/domain errors
- `EndpointError` / `WrapEndpointError` — API endpoint errors

### Merge Methods Added
- `Merge(errors ...AppError)` — combine multiple errors into one
- `MergeWithCode(code, errors)` — merge with explicit error code

### Enum Extensions
- New path variants: E4016–E4020 in `apperrtype`
- Utility maps: `CodeToVariantMap`, `StringToVariantMap`

### Naming Convention
- Renamed `URLError` → `UrlError` for consistent casing across all specs

### Error Documentation Guideline
- Created mandatory process: every bug fix must be documented in `app-issues/` folder
- Template: error description, root cause, solution, prevention
- Purpose: prevent AI hallucination on previously solved problems

- `02-apperror-struct.md` — sections 2.2.2–2.2.7 added

- `readme.md` — updated examples
